# Codex Instructions

This repository is a React design system. Follow the Swiss design rules in `./rules/swiss-design.md` when proposing or editing tokens, components, layouts, stories, and documentation.

## Working Priorities

1. Preserve a clear token hierarchy: base tokens, semantic tokens, then component tokens.
2. Prefer restrained, systematic UI decisions over decorative styling.
3. Keep component APIs predictable, typed, accessible, and documented in Storybook.
4. Treat mobile behavior as a first-class design requirement, not an afterthought.
5. Avoid unrelated refactors while working on a focused design-system change.
